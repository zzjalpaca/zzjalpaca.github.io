Markdown 介绍

a

  • 轻量级标记语言，2004 年被 John Gruber 创造
  • 它允许人们“使用易读易写的纯文本格式编写文档”
  • Markdown 最重要的设计就是可读性
  • Markdown 可以选择性的转换为 HTML 文档格式
  • 很多网站目前都使用了 Markdown 或者其变种，例如Github等大型社区

我们使用 Markdown 做什么

  • 演讲稿
  • 个人笔记
  • 技术博客
  • 电子邮件

基本语法

  • 标题 #
  • 段落 直接写
  • 粗体和斜体
    • **粗体内容**
    • *斜体内容*
  • 无序列表和有序列表
    • - 无序列表内容
    • 1. 有序列表内容
  • 链接和图片
    • [链接标题](链接地址)
    • ![图片alt](图片地址，本地和线上均可)
  • 引用：> 引用内容
  • 代码引用：
    • 单行引用：`内容` ，内容两边分别用单个反引号包裹
    • 多行引用：```中间是多行内容```，内容两边分别用三个反引号包裹
  • 表格
    • 用的比较少，可以参考 PPT

    • 用来在 Sublime 中辅助编写 Markdown

    • Chrome 中的插件，用来辅助使用 Markdown 格式发送邮件的

终端使用

  • pwd（print working directory）
  • ls（list show） 列出目录
    • ls -a 可以查看隐藏文件或目录
  • mkdir（make directory）新建目录
  • cd（change directory） 切换目录
    • cd .. 回退到上一级目录
  • touch 文件名 新建文件
  • cat 查看文本文件内容
  • clear 清屏

Git 基本使用

  • git init [项目名称]
    • 如果指定了项目名称，git 会帮你自动创建一个目录，目录名就是项目名称，同时在该目录下生成一个隐藏文件 .git
    • 如果想在已有项目中初始化一个 git 仓库，就通过终端进入该目录，然后执行 git init 就会自动在当前项目下创建一个版本管理仓库
      • 也就是会多出一个隐藏文件 .git
  • git status 查看工作树状态
    • 该命令会把未添加到版本管理仓库的文件或改动的文件列出来
  • git add 指定文件 添加文件到暂存区
    • git add --all 将所有改动文件添加到暂存区中，不需要指定某个具体的文件
    • git add . 同 --all
  • git commit -m “提交日志” 将暂存区中的文件提交到本地仓库
    • 注意：提交的时候一定要记得写提交日志
  • git log 查看提交日志
  • git remote add origin 线上仓库地址
  • git push -u origin master 第一次用这个，以后直接用 git push 就可以了

使用 Github Page 部署自己专属域名

  1. 新建仓库：用户名.github.io
  2. git clone 仓库地址
  3. 将静态文件都添加到该仓库中
  4. git push 到云端
  5. 然后通过 你的用户民.github.io 来访问该网站，默认找 index.html 页面

给 用户名.github.io 配置别名

  1. 买一个域名
  2. 登陆到域名服务提供商的配置解析界面
  3. 添加一个 CNAME ，别名填一个二级域名名称，别名主机填写你的 用户名.github.io
  4. 然后在你的静态网站仓库根路径下添加一个文件 CNAME，里面写入你刚才在 dns 里面加的那个别名
  5. 然后将改动提交到线上仓库
  6. 接下来就可以通过自定义的别名来访问你托管的静态网站了

快速创建个人静态博客

  1. 安装 Node 环境
  2. 在终端的任意目录执行 npm install hexo-cli -g
  3. 在你自己指定的目录中执行该命令：hexo init 博客名称 该过程需要联网
  4. cd blog

详细请参考：https://hexo.io/zh-cn/docs/deployment.html
